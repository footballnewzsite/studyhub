CSIRT / Incident Response — Πλήρης Οδηγός

Το Computer Security Incident Response Team (CSIRT) είναι η ομάδα υπεύθυνη για την αντιμετώπιση και διαχείριση περιστατικών ασφάλειας πληροφορικής. Στόχος της είναι η έγκαιρη ανίχνευση, ανάλυση, περιορισμός και αποκατάσταση μετά από κυβερνοσυμβάντα.

1. Ρόλος και Αρμοδιότητες CSIRT

  • Παρακολούθηση και ανίχνευση περιστατικών (monitoring & detection)
  • Ανάλυση απειλών (threat analysis)
  • Ανταπόκριση και αποκατάσταση (response & recovery)
  • Συνεργασία με εθνικούς/διεθνείς φορείς (ENISA, CSIRT-CY, CERT-EU)
  • Εκπαίδευση προσωπικού και awareness

2. Incident Response Lifecycle (NIST SP 800-61r2)

  1. Preparation – Πολιτικές, IR plan, training, εργαλεία (SIEM, forensic kits)
  2. Detection & Analysis – Αναγνώριση περιστατικού, συλλογή logs, alerts
  3. Containment, Eradication, Recovery – Περιορισμός, καθαρισμός, αποκατάσταση
  4. Post-Incident Activity – Lessons learned, βελτίωση διαδικασιών

4. Preparation — Incident Response Plan

  • IR Policy εγκεκριμένη από Management
  • Defined escalation paths
  • Contact list (IT, Legal, Management, CSIRT-CY)
  • Incident severity matrix
  • IR Kits: forensic USBs, write blockers, collection scripts

Δείγμα Πολιτικής (απλό απόσπασμα)

Purpose: To ensure incidents are identified, analyzed, contained, and reported effectively.
Scope: Applies to all IT systems and data assets.
Incident Definition: Any event that jeopardizes confidentiality, integrity, or availability.

5. Detection & Analysis

Εργαλεία και ενδείξεις που βοηθούν στον εντοπισμό περιστατικών:

  • SIEM alerts (π.χ. Splunk, ELK, QRadar)
  • EDR logs (CrowdStrike, Defender for Endpoint)
  • Network traffic analysis με Wireshark / Zeek
  • Host logs – Windows Event Viewer, syslog

Παράδειγμα Ανίχνευσης με Splunk Query

index=security sourcetype=windows:security EventCode=4625 
| stats count by user, src_ip 
| where count > 10

6. Forensic Investigation

Memory Analysis (Volatility)

volatility -f memory.img imageinfo
volatility -f memory.img --profile=Win10x64_19041 pslist
volatility -f memory.img malfind > malware_report.txt

Disk Forensics (Autopsy / Sleuth Kit)

fls -r -m / -p evidence.dd > filelist.txt
icat evidence.dd 12345 > suspect_file.exe

YARA Scanning Example

rule Suspicious_PDF {
  meta: author="CSIRT"
  strings: $pdf = "%PDF" $url = /http:\/\/[A-Za-z0-9\.\-]+/
  condition: $pdf and $url
}
yara Suspicious_PDF.yar evidence/

7. Containment, Eradication & Recovery

  • Απομόνωση μολυσμένων συστημάτων
  • Καθαρισμός malware ή πλήρης επαναφορά
  • Ενημέρωση patches και passwords
  • Έλεγχος για persistence mechanisms
  • Επαλήθευση επαναφοράς (validation & monitoring)

8. Post-Incident Activities

  • Lessons Learned Meeting (εντός 7 ημερών)
  • Root Cause Analysis (τεχνικό και οργανωτικό)
  • Ενημέρωση IR playbooks / training materials
  • Reporting προς Management ή CSIRT-CY

9. Playbooks (παραδείγματα)

Είδος ΣυμβάντοςΒασικά Βήματα
PhishingIsolate email → Extract headers → Block domain → Awareness email
Malware InfectionDisconnect host → Acquire memory → Scan with YARA → Reimage
Data BreachNotify DPO → Contain exposure → Forensic review → Notify regulator

10. Tools Summary

  • Volatility – Memory analysis
  • Autopsy / Sleuth Kit – Disk forensic
  • Cuckoo Sandbox – Malware dynamic analysis
  • Wireshark / Zeek – Network investigation
  • Splunk / ELK – SIEM analysis

11. Exam Q&A

Q1.

Ποια είναι τα στάδια του Incident Response Lifecycle;

A: Preparation → Detection → Containment → Recovery → Lessons Learned.

Q2.

Τι κάνει το εργαλείο Volatility;

A: Ανάλυση μνήμης RAM για ανίχνευση κακόβουλων διεργασιών.

Q3.

Πότε γίνεται η αναφορά σε CSIRT-CY;

A: Εντός 24 ωρών από την ανίχνευση περιστατικού (ΚΔΠ 389/2020).

Q4.

Ποια εργαλεία χρησιμοποιούνται για forensic imaging;

A: FTK Imager, dd, Autopsy/Sleuth Kit.

Q5.

Τι είναι Playbook;

A: Προκαθορισμένα βήματα αντιμετώπισης συγκεκριμένου τύπου περιστατικών.

12. Exam Tips

  • Θυμήσου τη σειρά των IR σταδίων — είναι συχνή ερώτηση.
  • Συνδύασε ISO/NIST γνώσεις όταν περιγράφεις διαδικασία IR.
  • Να γνωρίζεις 2–3 εργαλεία ανά στάδιο (SIEM για detection, Volatility για analysis, Autopsy για disk).
  • Απάντησε concisely, με keywords: “Containment – isolate system”, “Recovery – validate integrity”.
