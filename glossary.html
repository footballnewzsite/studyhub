Cybersecurity & Networking Glossary

Βασικοί όροι και ορολογία που συναντώνται συχνά στις εξετάσεις — με αγγλική ονομασία και επεξήγηση στα ελληνικά.

Networking

LAN (Local Area Network): Τοπικό δίκτυο μικρής γεωγραφικής εμβέλειας (π.χ. γραφείο, σπίτι).
WAN (Wide Area Network): Δίκτυο μεγάλης εμβέλειας που συνδέει πολλαπλά LANs (π.χ. Internet).
IP Address: Μοναδικός αριθμός που προσδιορίζει μια συσκευή στο δίκτυο.
Private IP: Διεύθυνση για εσωτερικά δίκτυα (10.x.x.x, 192.168.x.x).
Public IP: Διεύθυνση που είναι ορατή στο Internet.
Subnet Mask: Ορίζει ποιο μέρος της IP δείχνει το δίκτυο και ποιο τον host.
Router: Συσκευή που προωθεί πακέτα μεταξύ διαφορετικών δικτύων.
Switch: Συσκευή που συνδέει συσκευές στο ίδιο LAN.
Firewall: Φιλτράρει την κυκλοφορία δικτύου βάσει κανόνων ασφαλείας.
DNS (Domain Name System): Μετατρέπει domain names σε IP addresses.
DHCP: Υπηρεσία που εκχωρεί αυτόματα IP διευθύνσεις.
TCP: Πρωτόκολλο αξιόπιστης επικοινωνίας με επιβεβαίωση (ACK).
UDP: Ανεξάρτητη, χωρίς επιβεβαίωση επικοινωνία (π.χ. streaming).
VLAN: Εικονικό LAN για λογικό διαχωρισμό δικτύων.
Packet Sniffing: Παρακολούθηση/ανάλυση δικτυακής κυκλοφορίας (π.χ. Wireshark).

Cybersecurity & Risk

Confidentiality: Διασφάλιση ότι μόνο εξουσιοδοτημένα άτομα έχουν πρόσβαση σε πληροφορίες.
Integrity: Διατήρηση ακρίβειας και πληρότητας δεδομένων.
Availability: Διαθεσιμότητα συστημάτων και υπηρεσιών όταν χρειάζονται.
Vulnerability: Αδύναμο σημείο που μπορεί να εκμεταλλευτεί επιτιθέμενος.
Threat: Δυνητικός κίνδυνος ή ενέργεια που μπορεί να προκαλέσει ζημιά.
Risk: Πιθανότητα απειλής να εκμεταλλευτεί μια ευπάθεια.
Malware: Κακόβουλο λογισμικό (virus, trojan, ransomware).
Phishing: Απόπειρα υποκλοπής στοιχείων μέσω ψεύτικων emails.
DoS / DDoS: Επίθεση άρνησης υπηρεσίας (overload servers).
Zero-Day: Ευπάθεια χωρίς διαθέσιμο patch.
Patch Management: Διαδικασία ενημέρωσης λογισμικού για ασφάλεια.
Encryption: Κρυπτογράφηση δεδομένων για προστασία απορρήτου.
Hashing: Μονοδρομική συνάρτηση για έλεγχο ακεραιότητας (SHA-256).
2FA / MFA: Επαλήθευση ταυτότητας με δεύτερο παράγοντα (π.χ. SMS code).

Incident Response & Forensics

Incident: Γεγονός που επηρεάζει ή απειλεί την ασφάλεια ενός συστήματος.
CSIRT: Computer Security Incident Response Team – υπεύθυνη ομάδα IR.
Incident Response Plan: Καθορισμένες διαδικασίες για αντιμετώπιση περιστατικών.
Containment: Περιορισμός διάδοσης περιστατικού.
Eradication: Εξάλειψη κακόβουλου λογισμικού ή προσβολής.
Recovery: Επαναφορά συστημάτων σε κανονική λειτουργία.
Root Cause Analysis: Ανίχνευση της πραγματικής αιτίας ενός περιστατικού.
Forensic Imaging: Αντιγραφή δεδομένων με σκοπό διατήρηση αποδεικτικών στοιχείων.
Chain of Custody: Καταγραφή χειρισμού αποδεικτικών για να διασφαλιστεί ακεραιότητα.
Threat Intelligence: Ανάλυση και συλλογή πληροφοριών για τρέχουσες απειλές.
Vulnerability Assessment: Έλεγχος για γνωστές ευπάθειες.
Penetration Testing: Προσομοιωμένη επίθεση για αξιολόγηση άμυνας.
SIEM: Security Information and Event Management – συλλογή και ανάλυση logs.
IOC (Indicator of Compromise): Ίχνος ή στοιχείο που υποδεικνύει παραβίαση (hash, IP, domain).
Playbook: Προκαθορισμένα βήματα για αντιμετώπιση συγκεκριμένων περιστατικών.

Standards & Compliance

ISO/IEC 27001: Πρότυπο για σύστημα διαχείρισης ασφάλειας πληροφοριών (ISMS).
ISO/IEC 27002: Κατευθυντήριες γραμμές ελέγχων ασφαλείας.
NIST SP 800-53: Πλαίσιο ελέγχων ασφάλειας από το NIST.
NIST Cybersecurity Framework: Πλαίσιο 5 λειτουργιών (Identify, Protect, Detect, Respond, Recover).
PDCA Cycle: Plan–Do–Check–Act — συνεχής βελτίωση ISMS.
KΔΠ 389/2020: Κυπριακή απόφαση που καθορίζει μέτρα κυβερνοασφάλειας για κρίσιμες υποδομές.
