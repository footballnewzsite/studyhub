Πλήρης συλλογή εξεταστικών ερωτήσεων για τα κεφάλαια: Networks, Standards, Cybersecurity, CSIRT/IR.

Section 1 — Networks

Q1. Ποια είναι τα 7 επίπεδα του OSI model και ποιος είναι ο ρόλος τους;

Α:
1. Physical – μεταφορά bits μέσω καλωδίου/σήματος
2. Data Link – MAC addresses, frames, error detection (Ethernet)
3. Network – IP addressing, routing (IP, ICMP)
4. Transport – TCP/UDP, ports, reliability
5. Session – διαχείριση συνεδριών (API sessions)
6. Presentation – encryption, compression
7. Application – HTTP, DNS, SMTP, FTP.

Παράδειγμα: όταν ανοίγεις ένα site, το HTTP λειτουργεί στο Application layer, ενώ το TCP στο Transport.

Q2. Τι διαφορά έχει το TCP από το UDP;

Α: Το TCP (Transmission Control Protocol) είναι connection-oriented, εξασφαλίζει αξιόπιστη μετάδοση και επιβεβαίωση πακέτων (ACKs). Το UDP (User Datagram Protocol) είναι connectionless, χωρίς επιβεβαίωση — γρηγορότερο αλλά λιγότερο αξιόπιστο. Παράδειγμα: TCP → web browsing, FTP. UDP → video streaming, DNS queries.

Q3. Τι είναι οι τοπολογίες δικτύου;

Α: Σχεδιαστικά πρότυπα σύνδεσης κόμβων:
  • Bus – όλοι οι κόμβοι σε ένα κοινό καλώδιο
  • Star – κόμβοι συνδεδεμένοι σε switch (πιο συχνή)
  • Ring – δεδομένα κυκλικά
  • Mesh – πολλαπλές συνδέσεις (fault tolerance)

Section 2 — Security Standards

Q4. Ποια είναι τα βασικά σημεία του ISO/IEC 27001;

Α: Το ISO 27001 καθορίζει ένα Information Security Management System (ISMS) με τα εξής:
  • Risk Assessment & Treatment
  • Security Policies
  • Access Control, Cryptography, Incident Management
  • PDCA κύκλος (Plan–Do–Check–Act)

Q5. Τι είναι η Απόφαση ΚΔΠ 389/2020;

Α: Κυπριακή νομοθεσία για την κυβερνοασφάλεια κρίσιμων υποδομών. Καθορίζει υποχρεώσεις οργανισμών για:
  • Αναφορά περιστατικών σε CSIRT-CY
  • Εφαρμογή ISO 27001 / NIST standards
  • Ορισμό Υπευθύνου Ασφάλειας Πληροφοριών

Q6. Ποιες είναι οι 5 λειτουργίες του NIST Cybersecurity Framework;

Α: Identify, Protect, Detect, Respond, Recover. Παράδειγμα: Detect = SIEM monitoring, Respond = Incident Response plan.

Section 3 — CSIRT / Incident Response

Q7. Ποια είναι τα στάδια του Incident Response Lifecycle;

Α: Preparation → Detection & Analysis → Containment, Eradication & Recovery → Post-Incident Activity. Tip: αποστήθισε τη σειρά — είναι συχνή ερώτηση.

Q8. Ποια εργαλεία χρησιμοποιούνται στο IR;

  • Volatility – Memory analysis
  • Autopsy – Disk forensic
  • Cuckoo Sandbox – Malware analysis
  • SIEM – Detection & correlation
  • Wireshark – Network capture

Q9. Πώς συλλέγεις αποδεικτικά δεδομένα;

Α: Με τη μέθοδο forensic acquisition — δηλαδή αντιγραφή χωρίς αλλοίωση: 
dd if=/dev/sda of=evidence.dd bs=4M conv=noerror,sync
και καταγραφή hash με sha256sum. Σημαντικό: πάντα κρατάς chain of custody log.

Q10. Ποια είναι η διαφορά Threat Intelligence και Vulnerability Assessment;

Α:
  • Threat Intelligence: ανάλυση εξωτερικών απειλών, TTPs, IOCs, MITRE ATT&CK.
  • Vulnerability Assessment: εσωτερικός έλεγχος για αδύναμα σημεία (Nessus, OpenVAS).

Q11. Πότε γίνεται “lessons learned” meeting;

Α: Μετά το recovery — συνήθως εντός 7 ημερών. Στόχος: καταγραφή root cause και βελτίωση IR plan.

Section 4 — Practical Scenarios

Scenario 1: Phishing Incident

Βήματα:
  1. Isolate suspicious email
  2. Extract headers με Outlook Message Header Analyzer
  3. Check URLs με urlscan.io
  4. Notify users & block domain

Scenario 2: Ransomware Infection

  1. Disconnect infected host
  2. Acquire memory & disk image
  3. Identify ransomware note and hash sample
  4. Restore from offline backup
  5. Report to CSIRT-CY
💡 Exam Tip: Να μπορείς να περιγράψεις για κάθε τύπο περιστατικού: detection → containment → recovery → reporting.
