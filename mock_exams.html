Mock Exams — Επιλεγμένες ερωτήσεις με απαντήσεις

1–10: Networks & Protocols

Q1. Τι είναι το TCP 3-way handshake;

A: SYN → SYN-ACK → ACK. Ο client στέλνει SYN, ο server απαντά SYN-ACK, ο client κλείνει με ACK — έτσι γίνεται η εγκατάσταση της TCP σύνδεσης. Σημείο αξιολόγησης: γνώση flags, πόσο χρόνο κρατάει, και τι δείχνει ένα SYN flood (πληθώρα SYN χωρίς ολοκλήρωση).

Q2. Πότε χρησιμοποιούμε UDP αντί TCP;

A: Όταν απαιτείται χαμηλή καθυστέρηση & όχι υποχρεωτική αξιοπιστία — π.χ. VoIP, DNS queries, video streaming. Επιπλέον, UDP μπορεί να χρησιμοποιηθεί για amplification attacks (DNS amplification).

Q3. Πώς υπολογίζεις subnets για ένα δίκτυο /24 που θέλεις 8 subnets;

A: Χρειάζεσαι 3 bits (2^3=8) -> νέος prefix /27 (24 + 3). Κάθε /27 έχει 32 addresses (30 usable).

Q4. Τι είναι ARP spoofing;

A: Ο επιτιθέμενος δηλώνει ψευδώς στις συσκευές ότι η MAC του αντιστοιχεί σε IP άλλου host (π.χ. gateway), επιτυγχάνοντας MITM. Αντιμετώπιση: static ARP entries, DHCP snooping, dynamic ARP inspection.

Q5. Ποια είναι η διαφορά stateful vs stateless firewall;

A: Stateless firewall εξετάζει πακέτα μεμονωμένα (rules πάνω σε IP/port). Stateful firewall παρακολουθεί τις συνδέσεις και διατηρεί κατάσταση sessions (π.χ. επιτρέπει απαντήσεις μόνο σε established connections).

Q6. Τι είναι NAT και ποια προβλήματα δημιουργεί για forensics;

A: NAT μεταφράζει private IPs σε public — server logs βλέπουν μόνο τη public IP, δυσκολεύεται ο εντοπισμός συγκεκριμένου host πίσω από NAT χωρίς επιπλέον logs (NAT table, timestamps).

Q7. Εξηγείστε το concept VLAN;

A: VLANs διαχωρίζουν λογικά την κυκλοφορία στο layer 2. Ορισμός: κάθε VLAN είναι ένα broadcast domain. VLAN trunking (802.1Q) επιτρέπει πολλαπλά VLANs σε ένα φυσικό σύνδεσμο.

Q8. Πότε βλέπεις fragmented IP packets; τι πρέπει να γνωρίζεις;

A: Όταν το MTU του μονοπατιού είναι μικρότερο από το μέγεθος του IP packet — fragment reassembly μπορεί να χρησιμοποιηθεί από attackers για evasion. Παρακολούθηση και σωστή MTU setting/PMTU helps.

Q9. Τι είναι DNS cache poisoning;

A: Εισαγωγή ψευδών DNS records σε cache resolver, ώστε να ανακατευθύνει clients σε malicious IPs. Mitigations: DNSSEC, use of authenticated resolvers.

Q10. Ποια εργαλεία χρησιμοποίησες για network analysis;

A: Wireshark (packet analysis), tcpdump, nmap (scanning), netcat (tcp testing), traceroute, ss/netstat.

11–20: Security Principles & Standards

Q11. Τι είναι το ISMS;

A: Information Security Management System — σύστημα διαχείρισης ασφάλειας πληροφοριών, περιλαμβάνει πολιτικές, διαδικασίες, ρόλους, risk assessment, controls βάσει ISO 27001.

Q12. Εξηγήστε PDCA στο πλαίσιο ISMS;

A: Plan: ορισμός scope & risk assessment. Do: εφαρμογή controls. Check: audits, monitoring, vulnerability scans. Act: διορθώσεις & βελτιώσεις.

Q13. Ποια είναι μερικά σημαντικά controls του Annex A (ISO 27001);

A: Access control (A.9), Cryptography (A.10), Operations security (A.12), Business continuity (A.17), Supplier relationships (A.15).

Q14. Τι είναι NIST SP 800-53;

A: Κατάλογος ελέγχων για κυβερνητικά συστήματα, οργανωμένος σε families (AC, IA, AU, IR, SI κ.ά.) — χρησιμεύει ως blueprint για τεχνική εφαρμογή controls.

Q15. Πώς θα χαρτογραφήσεις ISO Annex A σε NIST controls;

A: Παράδειγμα: ISO A.9 (Access Control) ↔ NIST AC-1/AC-2/AC-17. Χάρτο με πίνακα για συχνές αντιστοιχίσεις (use vendor mapping documents).

Q16. Τι προβλέπει η ΚΔΠ 389/2020 (σύντομα);

A: Απαιτεί οργανωτικά & τεχνικά μέτρα, risk assessments, και notification στην Αρχή για σημαντικά incidents — εφαρμόζεται σε συγκεκριμένους φορείς (ελέγξτε κείμενο για λεπτομέρειες).

Q17. Τι είναι RTO και RPO;

A: RTO (Recovery Time Objective) = πόσο γρήγορα πρέπει να ανακάμψει υπηρεσία. RPO (Recovery Point Objective) = πόσο δεδομένα είναι αποδεκτό να χαθούν (παράδειγμα: 1 hour RPO).

Q18. Πώς μετράς MTTR;

A: Mean Time To Repair/Restore — average time to remediate incidents. Μειώνεται με αυτοματισμούς, καλά σχέδια IR και DR tests.

Q19. Τι είναι risk acceptance vs risk mitigation;

A: Acceptance = αποδοχή του κινδύνου χωρίς ενέργεια (συνήθως μικρή πιθανότητα/impact). Mitigation = εφαρμογή controls για μείωση πιθανότητας/impact.

Q20. Τι είναι third-party risk;

A: Κίνδυνος που σχετίζεται με προμηθευτές/outsourcers — απαιτεί due diligence, SLA security clauses, vendor assessments.

21–35: CSIRT / IR / Forensics

Q21. Περιγράψτε σύντομα το IR playbook για data exfiltration alert.

A: (1) Triage: verify SIEM alerts, check netflow/PCAP; (2) Contain: isolate hosts, block IPs; (3) Forensic capture: disk & memory image, logs; (4) Eradicate: remove backdoors; (5) Recover & notify; (6) Lessons learned.

Q22. Τι συλλέγεις σε live response;

A: Running processes, network connections (netstat/ss), open files, user sessions, WMI/registry info, volatile memory dump (if necessary and planned).

Q23. Πώς παίρνεις forensic disk image;

A: Use write-blocker or boot from forensic media, create bit-for-bit image with dd/FTK Imager, compute MD5/SHA256 checksum for source & image.

Q24. Τι είναι timeline analysis;

A: Συγκεντρώνεις timestamps από logs, MFT, registry, browser artifacts και κατασκευάζεις χρονολόγιο (tools: Plaso/log2timeline, Timesketch).

Q25. Ποια είναι τα βασικά στοιχεία στο forensic report;

A: Methodology, tools & versions, hashes, findings (IOCs), timeline, conclusions, evidence chain of custody.

Q26. Τι είναι MITRE ATT&CK; πώς το χρησιμοποιείς;

A: Knowledge base of adversary tactics & techniques. Χαρτογράφησε παρατηρούμενα TTPs σε ATT&CK για detection & hunting rules.

Q27. Διαφορά vulnerability assessment vs penetration test;

A: VA = inventory & scanning — list of vulnerabilities (non-exploitative). PenTest = exploit to demonstrate real impact (scoped, authorized).

Q28. Τι είναι IOC και πώς το μοιράζεσαι;

A: Indicator of Compromise: IPs, domains, file hashes. Share via TI feeds (STIX/TAXII), vendor portals, or internal TI platforms.

Q29. Ποια logs είναι σημαντικά για investigation;

A: Endpoint logs (EDR), Windows Event Logs, syslog, proxy logs, firewall logs, application logs, netflow/PCAP.

Q30. Πώς διεξάγεις malware static analysis;

A: Hashing, strings, PE header inspection, imports, packers; use tools: VirusTotal, PEStudio, Ghidra for reverse engineering.

Q31. Τι είναι EDR; γιατί είναι σημαντικό;

A: Endpoint Detection & Response — agent-based monitoring for detection, response (process trees, rollback, isolation). Επιτρέπει γρήγορη παρακολούθηση & containment.

Q32. Τι ενέργειες κάνεις μετά από phishing incident;

A: Identify affected users, reset creds, scan for web shells/malware, block malicious sender/domain, notify users, run phishing awareness training.

Q33. Πώς εκτιμάται impact data breach;

A: Identify type of data (PII, financial, IP), number of records, sensitivity; legal & regulatory obligations (notification thresholds).

Q34. Τι περιλαμβάνει threat hunting session;

A: Hypothesis generation, collect relevant logs, use TI & ATT&CK mappings, search for anomalous behaviors, create detections.

Q35. Πώς χειρίζεσαι evidence custody;

A: Document all handlers, timestamps, storage location, maintain write protection, compute & store hashes, sign chain of custody forms.

36–50: Practical & Policies

Q36. Δώστε policy example για Access Control (κύρια σημεία);

A: Purpose, Scope, Roles & responsibilities, Authentication (MFA), Authorization (RBAC), Account provisioning & deprovisioning, periodic review, logging & monitoring.

Q37. Τι είναι backup rotation (3-2-1 rule);

A: 3 copies of data, on 2 different media, 1 offsite. Έτσι μειώνεται ο κίνδυνος απώλειας από καταστροφή ή ransomware.

Q38. Πώς γράφεις post-incident report;

A: Executive summary, timeline, root cause, remediation steps, affected systems, data impact, recommendations, lessons learned, attachments (hashes, logs).

Q39. Τι περιλαμβάνει vulnerability management lifecycle;

A: Discover assets, scan, prioritize by risk/impact, patch/mitigate, validate, report & metrics.

Q40. Πώς σχεδιάζεις tabletop exercise;

A: Define scenario (e.g. ransomware), participants, injects, objectives, measure response timeline, conduct debrief, update playbooks.

Q41. Τι είναι SIEM correlation rule;

A: Συνδυασμός πολλαπλών events για να δημιουργήσει alert (π.χ. login failure spikes + new process execution => potential compromise).

Q42. Τι περιλαμβάνει penetration testing report;

A: Scope, methodologies, findings ranked by severity, proof-of-concept, remediation guidance, evidence (screenshots, commands).

Q43. Πώς ανιχνεύεις lateral movement;

A: Monitor for unusual SMB/WinRM/RDP activity, new admin tools launched, account usage from new hosts; use EDR telemetry & netflow.

Q44. Τι είναι data exfiltration και πώς τη σταματάς;

A: Exfiltration = unauthorized data transfer out. Stop: block channels (FTP, cloud uploads), DLP controls, network segmentation, block suspicious destinations.

Q45. Πώς ελέγχεις για persistence mechanisms;

A: Look for autoruns, scheduled tasks, services, suspicious registry entries, WMI persistence, DLL side-loading.

Q46. Τι είναι YARA rule;

A: Pattern matching rules to identify malware families based on strings/bytes/metadata — use in memory/disk scans.

Q47. Πώς δημιουργείς detection για Ransomware;

A: Monitor for mass file renaming, rapid file writes, high CPU/ciphering processes, creation of ransom notes — create SIEM correlations & EDR rules.

Q48. Ποια είναι τα κρίσιμα στοιχεία σε forensic timeline;

A: File modified/created timestamps, process execution times, login times, network connections, scheduled tasks, registry changes.

Q49. Πώς αξιολογείς exfiltrated data impact;

A: Identify type of data, regulatory impact (GDPR), customers affected, business impact, reputation & financial costs.

Q50. Τι πρέπει να περιλαμβάνει μια προετοιμασία CSIRT playbook;

A: Contact list, escalation matrix, IR steps per incident type, forensic procedures, communication templates (internal/external), testing schedule.
